Tech Stack

Features

User

  • Register and login with JWT authentication (passwords hashed with Werkzeug)
  • Browse parking lots — live search, availability filter, price sort
  • Book a spot — auto-assigns next available spot in chosen lot
  • Live duration timer and estimated cost on active bookings
  • Reservation receipt modal with full bill summary after checkout
  • Booking history — active and completed sessions in one view
  • Personal analytics — monthly spending chart, weekly usage pattern, quick stats
  • Export full booking history to CSV
  • Profile page — update name, email, username, change password
  • Monthly activity report delivered by email (Celery scheduled task)

Admin

  • Separate admin login with role-based JWT validation
  • Dashboard stat cards — total lots, users, active sessions, total revenue
  • Create, edit, delete parking lots with spot count management
  • Monitor all reservations — filter by active / completed, cancel active bookings
  • User management — view all users, total spend, current parking status
  • Export all data to CSV (async via Celery)
  • Analytics page — revenue trend, booking status, lot utilisation, daily bookings
  • Monthly revenue report delivered by email (Celery scheduled task)

System

  • Toast notification system — non-blocking success / error / warning messages
  • Redis-cached parking lot API responses (30s TTL, invalidated on booking)
  • Celery Beat scheduled tasks — daily reminders (6 PM), monthly admin report (1st, midnight), monthly user report (1st, 9 AM)
  • Seed data on first run — 5 demo users, 5 parking lots, 80+ past reservations, 5 active sessions

Live Deployment Limits

This app is hosted on Render's free tier. The following limitations apply:

FeatureStatusReason
Login, booking, dashboard Works Core Flask app runs normally
Analytics & charts Works Powered by Chart.js on the frontend
CSV export Works Generated inline (no Celery needed)
Email reminders Disabled Celery workers not available on free tier
Monthly reports Disabled Celery Beat scheduler not running
Data persistence Limited SQLite resets on redeploy — seed data auto-repopulates
Cold start ~30s App sleeps after 15 min inactivity on free tier
All limitations are free-tier specific. Full functionality is available when self-hosted via Docker Compose on a VPS.
